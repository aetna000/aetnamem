# AetnaMem 0.4.0 — Evidence to approved change

Released: 2026-07-20  
Status: stable package release; decision SDK remains explicitly experimental

AetnaMem 0.4.0 adds a provider-neutral collaborative decision kernel and an
Evidence-to-Decision (EtD) profile. It connects evidence, panel judgments,
recommendations, institutional approvals, scoped authorization, and guarded
implementation without coupling the library to one user system or model
provider.

## Why this release

Organizations rarely fail because they have no evidence. They fail at the
handoffs: which evidence informed a judgment, who was eligible to vote, how a
recommendation was adopted, who approved resources, and whether the action
that ran was exactly the action authorized.

This release makes those transitions explicit, versioned, auditable, and
host-embeddable.

## What is included

- Generic decision cases with pinned templates, immutable artifact revisions,
  exact evidence links, roles, capabilities, conflicts, and recusal.
- Frozen ballot eligibility, hidden-until-close voting, immutable vote
  revisions, deterministic outcomes, idempotency, and concurrency controls.
- Separate recommendation adoption, institutional approval, scoped change
  authorization, and guarded action execution.
- SQLite for local/single-server use and PostgreSQL for horizontal,
  multi-process hosts.
- Ed25519 and host-supplied AWS KMS principal attestations plus signed outcome,
  adoption, approval, authorization, and purge receipts.
- Decision-payload and conflict-of-interest retention policies with
  digest-bound logical purge receipts.
- Clinical and generic EtD templates, Markdown reporting, an offline verifier,
  a complete playground, and an external pilot/methodology-review runbook.
- No change to default `Memory` or memory-only MCP behavior. Claude, Grok/xAI,
  OpenAI, Ollama, conventional software, or non-LLM hosts can use the same API.

## Install

One command installs the complete package, including SQLite, PostgreSQL, and
asymmetric signing support:

```bash
pip install aetnamem
```

The host chooses the backend at runtime:

```python
from aetnamem.decisions import DecisionEngine

sqlite_engine = DecisionEngine("organization.db")
postgres_engine = DecisionEngine.postgres(database_dsn)
```

Run and verify the complete workflow:

```bash
aetnamem-etd-playground --db ./etd.db --output ./etd-output
aetnamem-etd-verify ./etd-output/decision-bundle.json
```

## Validation

The implementation passed 173 tests, including simultaneous PostgreSQL voter
processes, a vote-versus-close race, complete EtD-to-authorization flows,
signed receipt verification, retention purge, Python 3.10–3.13 CI, clean
wheel/sdist installation, and the existing 33/33 memory benchmark.

## Boundaries

The collaborative decision SDK remains experimental even though 0.4.0 is the
latest stable package version. AetnaMem does not claim GRADE compliance,
clinical validity, regulatory compliance, cryptographic secret ballots, or a
completed external pilot. A real institution and independent methodologist
must generate that evidence using the supplied pilot and review protocol.

See the [decision workflow specification](../decision-workflow-spec.md),
[host integration guide](../decision-host-integration.md), and [pilot and
methodology-review runbook](../etd-pilot-methodology-review.md).
