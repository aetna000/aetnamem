# AetnaMem 0.5.1 — Search memory by meaning, then follow the evidence

Status: **public patch release**

> **AetnaMem remembers whether remembering actually helped.**

AetnaMem 0.5.1 turns the evidence already stored beneath agent memory into an
investigation surface. An auditor can begin with ordinary words instead of an
internal memory ID, find matching memories and events, and reconstruct a
chronological trace from admission through context, action, and outcome.

This is a compatible patch release. Existing Python, CLI, default 15-tool MCP,
four-memory runtime, SQLite, and OpenClaw behavior remain available. Agent
recall is unchanged unless an operator explicitly uses the new investigation
commands.

## Install

The package name and normal installation path have not changed:

```bash
python3 -m pip install --upgrade aetnamem
```

For OpenClaw:

```bash
openclaw plugins install npm:openclaw-memory-aetnamem@latest --pin
```

Python `0.5.1` is paired with the previously prepared OpenClaw plugin `0.3.0`.

## Added

- `aetnamem memories` for a readable list of retained memory.
- `aetnamem search` for lexical investigation across memories, episodes,
  retrievals, audit events, runtime runs, and guarded actions.
- `aetnamem trace` for relationship expansion into one chronological story.
- Text or JSON output to stdout or `--output FILE`.
- Optional semantic and hybrid investigation modes backed by a derived SQLite
  vector sidecar.
- Ollama, OpenAI-compatible, sentence-transformers, and dependency-free
  diagnostic hashing embedding adapters.
- Versioned semantic-index epochs with exact cosine ranking and explainable
  reciprocal-rank fusion.
- `--audit-access` and `aetnamem access-log` for a separate digest-only,
  hash-chained record of investigator searches.
- A reproducible semantic-search benchmark harness and labeled golden cases.

## Integrity and scale

- Every semantic candidate is revalidated against canonical subject, status,
  and content digest before it can be returned.
- Stored vector dimensions must match the active epoch. Corruption raises an
  integrity error instead of silently truncating a dot product.
- Verification reports are cached by canonical-record and vector-index
  generations maintained by SQLite triggers.
- Canonical records are fetched in batches instead of one query per vector.
- Larger exact searches use NumPy vectorization when NumPy is already
  available, with a dependency-free Python fallback.
- Ollama epochs pin the model SHA-256 digest reported by `/api/tags` and fail
  closed if the local model weights change.

Exact vector search remains linear in record count and dimensions. These
changes remove repeated verification and N+1 database work; they do not claim
sublinear retrieval.

## Deletion

When a registered semantic sidecar exists, forgetting a memory removes its
vectors from every epoch, verifies absence, marks affected epochs dirty, and
returns `aetnamem-deletion-receipt-v2` with cleanup and verification digests.
Without a sidecar, the existing v1 receipt remains unchanged.

The receipt describes the live canonical database and registered sidecar. It
does not claim erasure from backups, filesystem snapshots, process memory,
swap, or storage-device remnants.

## Access-audit boundary

Investigation commands do not create agent recall events or modify the agent's
behavioral evidence chain. Access recording is explicit:

```bash
aetnamem search ./memories.db "departure location" \
  --subject user-1 \
  --mode hybrid \
  --audit-access \
  --access-actor auditor@example.test

aetnamem access-log ./memories.db --subject user-1
```

The access chain stores digests rather than raw query or result content.
`--access-actor` is caller-supplied metadata, not authentication; a multi-user
service must provide trusted identity and authorization.

## Compatibility

- `pip install aetnamem` remains the installation command.
- Semantic model dependencies are optional.
- Existing `Memory` calls and the default MCP catalog are unchanged.
- Semantic search is an investigator tool, not a replacement for governed
  agent recall.
- OpenClaw `MEMORY.md` files and existing plugin configurations are not
  rewritten.
- The four-memory runtime and Causal Memory Ledger remain opt-in; causal
  improvement is not yet claimed.
